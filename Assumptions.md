# Assumptions

## Environment Setup

- The application is assumed to be running successfully with a React.js frontend and a Spring Boot backend.
- The application is accessible through the following routes:
  - `/register` – User Registration
  - `/login` – User Login
  - `/dashboard` – Protected Dashboard
- Selenium tests are executed using Google Chrome with Python, Selenium WebDriver, and ChromeDriver installed.

---

## JWT Behavior

- A JWT token is generated by the backend after a successful login.
- The JWT token is stored by the application (e.g., in local storage or a secure cookie) and is included in requests to access protected resources.
- Users without a valid JWT token are redirected to the Login page when attempting to access the Dashboard.

---

## Token Expiration

- JWT tokens are assumed to have a fixed expiration time (e.g., 1 hour).
- If the token expires, the user must log in again to obtain a new JWT token.
- Requests made with an expired or invalid token are rejected with an authentication error.

---

## Validation Rules

- Both the frontend and backend validate user inputs.
- Email addresses are validated for proper format, converted to lowercase, and must be unique.
- The Name field accepts only 5–24 characters.
- Passwords must contain at least 12 characters and include both letters and numbers.
- The Confirm Password field must exactly match the Password field before registration is allowed.
